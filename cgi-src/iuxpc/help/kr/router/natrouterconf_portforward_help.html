• 포트포워드 설정
    • 공유기의 외부 네트워크 사용자가 내부 네트워크에 접속하고자 하는 경우 사용되는 어플리케이션의 포트번호를 내부 네트워크의 IP 주소와 내부포트 번호로 연결시켜주는 기능이 필요한데, 이런 기능을 포트포워드라고 합니다.
    • 포트포워딩은 사용자가 직접 정의한 규칙과 UPNP에 의해 자동으로 설정되는 규칙으로 구분할 수 있으며, 사용자 정의 규칙이 UPNP에 의한 규칙보다 우선권이 높습니다.
    • UPNP에 의해 자동으로 설정되는 기능은 [시스템관리]-[기타설정]-[UPNP설정]에서 제어할 수 있습니다.
    • 포트포워드 + 트리거 규칙을 합하여 최대 100개 까지 설정 가능합니다.

  • 규칙이름
    • 임의로 입력가능하며 구분이 쉽도록 입력합니다.
  • 내부 IP 주소
    • 어플리케이션이 설치된 내부서버 또는 PC의 IP 주소를 입력합니다.
  • 프로토콜
    • Protocol Type (TCP/UDP/GRE/TCPUDP)을 선택 합니다.
  • 외부 포트
    • 외부 인터넷에서 접속시 사용되는 내부서버 또는 PC에 설치된 어플리케이션 포트범위를 설정합니다.
  • 내부 포트
    • 내부서버 또는 PC에 설치된 어플리케이션 포트범위를 설정합니다.
    • '외부 사용 포트 범위'와 동일한 경우에는 입력하지 않아도 됩니다.
  • 순위
    • 규칙이 동작할 우선 순위를 지정합니다. 입력하지 않는 경우, 자동으로 가장 낮은 순위로 규칙이 추가됩니다.

  • 포트트리거 설정
    • 어플리케이션의 온라인 접속시 사용되는 포트번호를 감지하여, 어플리케이션 사용시 필요한 포트를 현재 사용중인 PC로 자동 포트 포워딩을 시켜주는 기능.

  • 규칙이름
    • 임의로 입력가능하며 구분이 쉽도록 입력합니다.
  • 트리거 조건
    • 어플리케이션이 사용하는 Protocol Type (TCP/UDP)을 선택 합니다.
    • 어플리케이션 온라인 접속을 감지하기 위한 포트범위를 입력합니다.
  • 포트포워드
    • 어플리케이션이 사용하는 Protocol Type (TCP/UDP)을 선택 합니다.
    • 어플리케이션 사용을 위해 포트포워딩이 필요한 포트들을 입력 합니다. 포트입력시 구분은 '공백'으로 합니다. (예시)8080 8081 8082
    • 최대 10개의 포트까지 입력 가능합니다.
  • 순위
    • 규칙이 동작할 우선 순위를 지정합니다. 입력하지 않는 경우, 자동으로 가장 낮은 순위로 규칙이 추가됩니다.
