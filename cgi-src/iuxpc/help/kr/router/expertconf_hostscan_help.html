• ICMP PingTest
    • ICMP 스캔은 Windows의 Ping 명령어와 같은 원리로, 네트워크에 존재하는 상대(PC 또는 네트워크 장비)에게 회신요청의 메세지를 전달하고, 이에대한 응답을 받음으로써 상대의 상태를 파악할 수 있는 기능입니다.
    • 사용할 수 있는 옵션으로는 상대방의 IP주소, 회신요청 횟수, 응답제한 시간, 그리고 회신요청 메세지의 크기(Byte)가 있습니다.
    • 요청에대한 응답으로는 다음과 같은 메세지들이 있습니다.
    • Reply from : 상대의 응답 정보 표시
    • Request timed out : 응답제한 시간초과 또는 응답 불가함
    • Host Unreachable : 상대가 네트워크에 존재하지 않거나, 전원이 꺼져있는 등의 문제로 응답이 불가, 또는 본 공유기의 디폴트 게이트웨이가 설정되어 있지 않음.
    • Network Unreachable : 해당 네트워크 경로를 찾을 수 없거나, 본 공유기의 디폴트 게이트웨이가 설정되어 있지 않음.
    • 만약 상대가 방화벽을 사용하는 환경에 있다면, 회신요청에 대하여 응답하지 않을 수도 있습니다.
  • ARP PingTest
    • 해당 IP로 ARP 요청을 보내 해당 IP가 네트워크에 연결이 되어 있는지를 확인 할 수 있습니다.
    • ICMP와 회신 메세지의 크기만을 제외 하고 같은 옵션을 사용 합니다.
    • Reply from : ARP 응답에 대한 정보
    • Request timed out : 제한 시간 초과 또는 응답이 없음
    • 방화벽과 상관없이 TEST를 할 수 있습니다.
  • TCP포트스캔
    • IP 주소 : 포트 스캔을 하기 위한 IP주소를 설정 할 수 있습니다.
    • 포트 범위 : 해당 IP 범위의 포트 범위를 설정 할 수 있습니다.예를 들어 21~23으로 설정할 경우에는 해당 IP의 21번 부터 23번 까지의 포트 상태를 보여줍니다.
